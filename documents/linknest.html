Project Overview

LinkNest is a modern "Link in Bio" web application that provides users with the ability to create personalized pages where they can organize all their important links. It offers functionality similar to Linktree but with a modern tech stack and enhanced features.

Key Features

✨ Core Functionality

  • Custom Handle Creation: Users can create their unique handle for personalized links
  • Multiple Links Management: Add unlimited links with custom text descriptions
  • Profile Customization: Add profile pictures and detailed descriptions
  • Responsive Design: Optimized for both mobile and desktop devices
  • Real-time Validation: Handle availability check instantly
  • Animated UI: Smooth animations using Framer Motion

🎨 Visual Features

  • Glass Morphism UI: Modern transparent/blur effects
  • Gradient Backgrounds: Beautiful purple-to-black gradients
  • Animated Elements: Floating particles, hover effects
  • Mobile Preview: Live mobile mockup during creation
  • Interactive Cards: Hover effects and smooth animations

📱 PWA Support

  • Offline Functionality: Works without internet
  • Service Workers: Caching for better performance
  • Installable: Can be installed like an app on mobile devices
  • Manifest File: Complete PWA configuration

API Endpoints

1. Link Creation API

POST /api/link

Purpose: Creates new link tree profiles with custom links

{
  "handle": "username",
  "links": [
    {
      "link": "https://example.com",
      "linktext": "My Website"
    }
  ],
  "pic": "profile_image_url",
  "des": "Profile description"
}

2. Handle Check API

GET /api/checkhandle

Purpose: Validates username availability for registration

Query Parameter: ?handle=username

Database Schema

MongoDB Collection: `links`

{
  _id: ObjectId,
  handle: String,          // Unique username
  links: [                 // Array of links
    {
      link: String,        // URL
      linktext: String     // Display text
    }
  ],
  pic: String,             // Profile image URL
  des: String,             // Profile description
  createdAt: Date,         // Creation timestamp
  updatedAt: Date          // Last update timestamp
}

# Start production server
